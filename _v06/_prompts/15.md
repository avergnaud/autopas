# Prompt 15 — Fix: xlsx "problème dans le contenu" à l'ouverture dans Excel

## Problème rapporté

Excel affiche au téléchargement du fichier rempli :
> "Désolé... Nous avons trouvé un problème dans le contenu de « PAS_rempli_proj_...xlsx »"

## Diagnostic

Analyse du fichier ZIP/XML corrompu. Toutes les entrées XML parsent sans erreur.
La cause est identifiée dans les worksheet XML (`xl/worksheets/sheet1.xml`) :

### Bug : openpyxl écrit `sqref=` (minuscule) dans `<dataValidation>`

openpyxl 3.x sérialise l'attribut de plage des validations de données avec le nom
`sqref` (tout minuscule). Or la spec OOXML (ECMA-376 §18.3.1.32) exige `sqRef`
(camelCase). XML est **case-sensitive** ; Excel ne reconnaît pas `sqref` comme
l'attribut attendu, considère la validation sans plage et affiche le dialogue de
réparation.

**Avant (bugué) :**
```xml
<dataValidation sqref="F32:F45 F47:F51 …" type="list">
  <formula1>Cotation</formula1>
</dataValidation>
```

**Après (corrigé) :**
```xml
<dataValidation sqRef="F32:F45 F47:F51 …" type="list">
  <formula1>Cotation</formula1>
</dataValidation>
```

### Note : `<selection sqref="…">` est correct

L'élément `<selection>` utilise `sqref` **minuscule** par spec (CT_Selection §18.3.1.76).
Il ne doit PAS être modifié — le fix est ciblé `<dataValidation>` uniquement.

### Observations supplémentaires (non bloquantes)

- `xl/styles.xml` contient 16 204 entrées `<cellStyle>` (bloat openpyxl après 2 load/save).
  Dans la limite Excel (65 490), donc pas la cause du dialog mais alourdit le fichier.
- Plusieurs `definedName` pointent sur `#REF!` (noms référençant des onglets supprimés
  par `delete_unlisted_sheets`). Excel gère normalement ces `#REF!` sans dialog de
  réparation.

## Correction apportée

Fichier modifié : `app/services/parser_xlsx.py`

Ajout de la fonction publique `fix_openpyxl_xml(filepath)` :
- Utilise `_xlsx_zip_replace` pour cibler les worksheet XML
- Regex `re.sub(r"<dataValidation\b[^>]*>", _fix_tag, text)` : remplace `sqref=`
  par `sqRef=` **uniquement** à l'intérieur des balises ouvrantes `<dataValidation>`
- Appel ajouté à la fin de `write_responses()` (après `wb.save()`)
- Appel ajouté à la fin de `delete_unlisted_sheets()` (après `wb.save()`, uniquement
  si des onglets ont été supprimés)
