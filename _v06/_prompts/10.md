# Prompt 10 — Anonymisation : définition des mots-clés et suppression des métadonnées

## Demande

Implémenter la première étape du wizard d'anonymisation :
1. Permettre à l'utilisateur de définir les mots-clés à anonymiser (avec suggestions extraites des métadonnées du fichier).
2. Anonymiser le fichier xlsx : remplacement des mots-clés dans les cellules + suppression des métadonnées (PII).

## Décisions techniques

### Nouveau service : `app/services/anonymizer.py`

- `extract_metadata(xlsx_path)` : lit `docProps/core.xml` (auteur, titre, sujet, …) et `docProps/app.xml` (société, application) directement en tant que ZIP, sans passer par openpyxl (plus fiable pour les champs optionnels).
- `anonymize_xlsx(source, dest, keywords)` :
  - Trie les mots-clés par longueur décroissante (évite les remplacements partiels, ex. "Ministère des Armées" avant "Armées").
  - Tokens : `[CONFIDENTIEL_1]`, `[CONFIDENTIEL_2]`, …
  - Préserve les named ranges locaux (dropdowns Excel) via `safe_local_defined_names()` — fonction migrée depuis `web.py`.
  - Nettoie `core_properties` via openpyxl (creator, last_modified_by, title, subject, …).
  - Patch `docProps/app.xml` via ZIP pour supprimer Company et Application (non exposé par openpyxl).
- Sauvegarde `anonymized_map.json` (mapping `{keyword: token}`) pour la dé-anonymisation future.

### Nouveaux endpoints dans `app/api/web.py`

| Méthode | URL | Rôle |
|---------|-----|------|
| GET | `/api/projects/{id}/metadata` | Extraire les suggestions de mots-clés |
| POST | `/api/projects/{id}/anonymize` | Appliquer l'anonymisation |
| GET | `/api/projects/{id}/anonymized` | Télécharger `anonymized.xlsx` |

### UI (`web/private.html`)

Wizard en 3 sections :
1. **upload-section** — upload du fichier xlsx
2. **keywords-section** — suggestions (checkboxes pre-cochées) + textarea pour mots-clés supplémentaires
3. **anonymized-section** — table de correspondance + lien de téléchargement

Après upload, `loadMetadata()` appelle `/api/projects/{id}/metadata` pour pré-remplir les suggestions.
