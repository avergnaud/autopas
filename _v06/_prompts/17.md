# Prompt 17 — Fix : Excel repair dialog sur `PAS_rempli_*.xlsx` (externalLinkLongPath rId mismatch)

## Erreur observée

Microsoft Excel affiche « Désolé... Nous avons trouvé un problème dans le contenu de `PAS_rempli_proj_*.xlsx` » à l'ouverture du fichier de sortie.

## Diagnostic

### Contexte

Le fichier d'entrée est un xlsx SharePoint. Il contient des **external links** (11 au total) dont
certains pointent vers d'autres classeurs sur SharePoint via la relation moderne
`externalLinkLongPath` (introduite dans Excel 2019 / M365) en plus du type classique
`externalLinkPath`.

### Cause racine — mismatch rId dans les rels d'external link

Pour chaque external link, le fichier `xl/externalLinks/externalLink{N}.xml` référence le
classeur externe via `<externalBook r:id="rId1">`. Le fichier de relation correspondant
`xl/externalLinks/_rels/externalLink{N}.xml.rels` doit donc contenir une entrée `Id="rId1"`.

**Avant traitement (fichier original)**, le rels contenait deux entrées :
```xml
<Relationship Id="rId1" Type=".../externalLinkPath"          Target="file:///..."/>  <!-- chemin local classique -->
<Relationship Id="rId2" Type=".../externalLinkLongPath" Target="https://sharepoint.com/..."/>
```

**Après sauvegarde openpyxl**, openpyxl supprime l'entrée `externalLinkPath` (chemin local)
qu'il ne sait pas gérer et ne conserve que `externalLinkLongPath` — mais **sous son ID
d'origine** `rId2` :
```xml
<Relationship Id="rId2" Type=".../externalLinkLongPath" Target="https://sharepoint.com/..."/>
```

Le XML reference toujours `r:id="rId1"` mais `rId1` n'existe plus dans le rels → Excel détecte
la référence brisée et affiche le dialogue de réparation.

**Conséquence** : externalLink1 et externalLink10 (les deux liens avec `externalLinkLongPath`)
avaient ce mismatch.

## Correction

Fichier modifié : `app/services/parser_xlsx.py`

### Nouvelle fonction `_fix_extlink_rels(filepath)`

Scanne les fichiers `xl/externalLinks/externalLink{N}.xml` pour lire l'`r:id` attendu.
Pour chaque rels file où l'ID attendu est absent et où il n'y a qu'un seul ID disponible,
renomme cet ID pour correspondre à l'attendu.

```python
def _fix_extlink_rels(filepath: Path) -> None:
    """Rename mismatched rId in external link rels files."""
    id_renames = {}
    with zipfile.ZipFile(filepath, "r") as zin:
        nameset = set(zin.namelist())
        for name in sorted(nameset):
            m = re.match(r"xl/externalLinks/externalLink(\d+)\.xml$", name)
            if not m:
                continue
            rels_path = f"xl/externalLinks/_rels/externalLink{m.group(1)}.xml.rels"
            if rels_path not in nameset:
                continue
            xml = zin.read(name).decode("utf-8")
            rels = zin.read(rels_path).decode("utf-8")
            ref = re.search(r'<externalBook\b[^>]+\br:id="([^"]+)"', xml)
            if not ref:
                continue
            expected_id = ref.group(1)
            actual_ids = re.findall(r'\bId="([^"]+)"', rels)
            if expected_id not in actual_ids and len(actual_ids) == 1:
                id_renames[rels_path] = {actual_ids[0]: expected_id}
    if not id_renames:
        return
    # Rewrite zip with corrected rels files
    ...
```

### Appel dans `fix_openpyxl_xml`

`_fix_extlink_rels` est appelé à la fin de `fix_openpyxl_xml`, après le fix `sqref → sqRef`.

## Résultat

Après le fix :
- `externalLink1` : rels `Id="rId1"` ↔ XML `r:id="rId1"` ✓
- `externalLink10` : rels `Id="rId1"` ↔ XML `r:id="rId1"` ✓
- `<dataValidation sqRef=…>` toujours correct ✓
- `<selection sqref=…>` toujours en minuscule (correct) ✓
