# Prompt 18 — Fix : Excel repair dialog sur `PAS_rempli_*.xlsx` (VML namespace prefixes)

## Erreur observée

Microsoft Excel affiche « Désolé... Nous avons trouvé un problème dans le contenu de `PAS_rempli_proj_*.xlsx` » à l'ouverture du fichier de sortie, malgré les fixes des prompts 15-17.

## Diagnostic

### Analyse exhaustive du fichier généré

Après avoir vérifié et confirmé que les deux bugs précédents (sqRef, externalLink rId) sont bien
corrigés dans le fichier `131708`, une analyse XML complète a révélé :

- Tous les fichiers XML sont valides (ET.fromstring passe) ✓
- Tous les indices de styles (fontId, fillId, borderId, dxfId, xfId) sont dans les bornes ✓
- Toutes les références OPC (relationships) pointent vers des fichiers existants ✓
- `xl/styles.xml` contient 16 204 `cellStyles` — large mais valid, indices bijectifs ✓
- **`xl/drawings/commentsDrawing1.vml` utilise des préfixes de namespace incohérents** ← BUG

### Cause racine — VML namespace prefix reuse

Quand openpyxl sauvegarde un classeur qui contient un dessin VML (commentaires), il réécrit
le fichier `.vml` avec des préfixes génériques (`ns0:`, `ns1:`, `ns2:`, `ns3:`) et **réutilise
le même préfixe pour des URIs de namespace différentes** dans différents éléments frères :

**Avant traitement (fichier original)**, le VML utilisait les préfixes standards :
```xml
<xml>
  <o:shapelayout ...>       <!-- o: = office namespace -->
  <v:shapetype ...>         <!-- v: = vml namespace    -->
  <v:shape ...>             <!-- v: = vml namespace    -->
    <x:ClientData ...>      <!-- x: = excel namespace  -->
  </v:shape>
</xml>
```

**Après sauvegarde openpyxl**, le VML devient :
```xml
<xml>
  <ns0:shapelayout xmlns:ns0="office" xmlns:ns1="vml" ...>  <!-- ns0=office -->
  <ns2:shapetype xmlns:ns2="vml" xmlns:ns3="office" ...>    <!-- ns2=vml    -->
  <ns0:shape xmlns:ns0="vml" xmlns:ns1="office" ...>        <!-- ns0=vml !! ns0 RÉUTILISÉ -->
    <ns2:ClientData xmlns:ns2="excel" ...>                  <!-- ns2=excel !! ns2 RÉUTILISÉ -->
  </ns0:shape>
</xml>
```

Le préfixe `ns0` est utilisé pour l'URI `office` dans `<ns0:shapelayout>` puis pour l'URI
`vml` dans `<ns0:shape>`. Idem pour `ns2`. Bien que techniquement valide en XML (les préfixes
sont locaux à leur scope), le parser VML d'Excel est sensible à cette incohérence et peut
déclencher le dialogue de réparation.

## Correction

Fichier modifié : `app/services/parser_xlsx.py`

### Nouvelle fonction `_fix_vml_namespaces(filepath)`

Re-parse chaque fichier `.vml` du zip avec `xml.etree.ElementTree` après enregistrement des
préfixes standards, puis re-sérialise de façon à hisser toutes les déclarations de namespace
à l'élément racine avec des préfixes cohérents et conventionnels.

```python
def _fix_vml_namespaces(filepath: Path) -> None:
    import xml.etree.ElementTree as ET

    _VML_NAMESPACES = {
        "v": "urn:schemas-microsoft-com:vml",
        "o": "urn:schemas-microsoft-com:office:office",
        "x": "urn:schemas-microsoft-com:office:excel",
    }
    for prefix, uri in _VML_NAMESPACES.items():
        ET.register_namespace(prefix, uri)

    def _normalize(text: str) -> str:
        root = ET.fromstring(text)
        return ET.tostring(root, encoding="unicode")

```

### Résultat de la normalisation

**Avant** (openpyxl) :
```xml
<xml>
  <ns0:shapelayout xmlns:ns0="office" xmlns:ns1="vml" ns1:ext="edit">
    <ns0:idmap ns1:ext="edit" data="1"/>
  </ns0:shapelayout>
  <ns2:shapetype xmlns:ns2="vml" xmlns:ns3="office" ...>
  <ns0:shape xmlns:ns0="vml" xmlns:ns1="office" ...>    ← ns0 réaffecté !
    <ns2:ClientData xmlns:ns2="excel" ...>              ← ns2 réaffecté !
```

**Après** (normalisé) :
```xml
<xml xmlns:o="office" xmlns:v="vml" xmlns:x="excel">
  <o:shapelayout v:ext="edit">
    <o:idmap v:ext="edit" data="1" />
  </o:shapelayout>
  <v:shapetype ...>
  <v:shape ...>
    <x:ClientData ...>
```

### Appel dans `fix_openpyxl_xml`
