# Prompt 16 — Fix : `DataValidation.__init__() got an unexpected keyword argument 'sqRef'`

## Erreur produite

```
TypeError: DataValidation.__init__() got an unexpected keyword argument 'sqRef'
  File "…/parser_xlsx.py", line 115, in write_responses
      wb = openpyxl.load_workbook(filepath)
```

## Diagnostic

La correction du prompt 15 (`fix_openpyxl_xml`) introduit un nouveau problème dans le pipeline :

### Chaîne d'événements

1. **Étape 1b** — `delete_unlisted_sheets(working_path)` :
   - openpyxl charge et sauvegarde le working file → écrit `sqref` (minuscule, bug openpyxl)
   - `fix_openpyxl_xml` est appelé → convertit `sqref` → `sqRef` dans le working file

2. **Étape 3** — `extract_questions(working_path, ...)` :
   - openpyxl charge en `read_only=True` → ne parse pas les dataValidations → pas d'erreur

3. **Étape 6** — `write_responses(output_path, ...)` :
   - `shutil.copy2(working_path, output_path)` → copie le working file (avec `sqRef`)
   - `openpyxl.load_workbook(output_path)` → parse les dataValidations → trouve `sqRef`
     comme attribut XML → le passe en kwarg → `DataValidation.__init__()` ne reconnaît
     pas `sqRef` (il n'accepte que `sqref` en minuscule) → **crash**

### Cause racine

`fix_openpyxl_xml` ne doit être appliqué qu'**une seule fois, sur le fichier de sortie final**,
après le dernier `wb.save()`. L'appeler sur le working file intermédiaire corrompt ce fichier
pour tous les chargements openpyxl ultérieurs (car openpyxl ne peut pas relire `sqRef`).

`write_responses` appelle déjà `fix_openpyxl_xml` à la fin (après `wb.save`) — c'est le bon
et unique endroit.

## Correction

Fichier modifié : `app/services/parser_xlsx.py`

Retrait de l'appel `fix_openpyxl_xml(filepath)` dans `delete_unlisted_sheets` :

```python
# Avant (bugué)
if to_delete:
    wb.save(filepath)
    fix_openpyxl_xml(filepath)   # ← convertit sqref→sqRef dans le working file
wb.close()

# Après (corrigé)
if to_delete:
    wb.save(filepath)            # ← working file conserve sqref (openpyxl peut relire)
wb.close()
```

`fix_openpyxl_xml` reste appelé uniquement dans `write_responses`, sur le fichier de sortie
final, après le dernier `wb.save()`.
