## Prompt 4

> I want to implement the following: if 'pas_niveau_entreprise' is true, then the json must not contain the following fields: 'nb_etp', 'expertise_atlassian', 'hebergement_donnees', 'cloud_provider', 'sous_traitance_rgpd', 'lieu_travail', 'agences', 'poste_travail', 'connexion_distante'. Because later on we will search for corpus xlsx files according to these metadatas. And I do not want to pollute the search. Do you think it makes sense?

Oui — ces champs sont spécifiques à une prestation. Un PAS niveau entreprise n'a pas de périmètre défini, donc ces données n'auraient pas de sens dans la recherche de similarité.

**`_v07/app/api/web.py`** — Bloc conditionnel dans `POST /api/corpus/{id}/metadata` :
```python
pas_entreprise = a.get("pas_niveau_entreprise") == "Oui"
meta = {
    **existing,
    "pas_niveau_entreprise": pas_entreprise,
    "type_prestation": _resolve_type_prestation(a),
    "activites": a.get("activites", ""),
    "secteur_client": a.get("secteur_client", ""),
    "date_remplissage": body.date_remplissage,
    "tags_supplementaires": body.tags,
}
if not pas_entreprise:
    meta.update({
        "nb_etp": int(a["nb_etp"]) if a.get("nb_etp") else None,
```
