# Prompt 13 — Détection de structure des questionnaires

## Contexte

Après l'anonymisation, l'application doit détecter automatiquement la structure du fichier xlsx (colonnes ID, question, réponse, statut, ligne d'en-tête) **via Claude API**, sur le fichier anonymisé (jamais sur l'original).

## Prompts de la conversation

### 1
> je pense qu'il y a une autre problématique : lorsque l'utilisateur upload un fichier (dans les 2 grands cas d'utilisation), notre application doit détecter dans ce fichier : la colonne qui correspond aux identifiants de questions, la colonne de questions, éventuellement une colonne de statut (liste déroulante), la colonne de réponses, la ligne de la première question… il me semble que notre code actuel ne gère pas cette problématique.

**Réponse :** Confirmation du gap. La spec prévoit F-PARSE-02 et F-WEB-04. Proposition d'insérer une étape "Structure" avec auto-détection via Claude. Nouveaux flux proposés :
- PAS : Upload → Anonymisation → Structure → Confirmation → Suite
- Corpus : Upload → Wizard → Anonymisation → Structure → Confirmation → Infos → Save

### 2
> l'étape "Structure" doit avoir lieu après l'étape "Anonymisation". En effet je ne veux pas envoyer à Claude la moindre donnée confidentielle ou personnelle.

**Réponse :** Validation de l'ordre. Implémentation complète.

## Fichiers créés / modifiés

| Fichier | Modifications |
|---|---|
| `app/services/structure_analyzer.py` | Nouveau — détection structure xlsx via Claude |
| `app/api/web.py` | 4 nouveaux endpoints structure (projects + corpus) + modèle `StructureModel` |

## Détail technique

### `structure_analyzer.py`
- Lit l'xlsx anonymisé (openpyxl, read-only)
- Extrait les 30 premières lignes de chaque onglet
- Formate un aperçu avec **lettres de colonnes Excel** (A, B, C…) pour que Claude réponde avec les bonnes lettres
- Appelle Claude (modèle configurable via `CLAUDE_MODEL`) avec prompt JSON strict
- Retourne : `selected_sheet`, `all_sheets`, `header_row`, `first_data_row`, `col_id`, `col_question`, `col_response`, `col_status`, `confidence`, `notes`
- Si `ANTHROPIC_API_KEY` absente/placeholder → `RuntimeError` → HTTP 503

### Endpoints
- `POST /api/projects/{id}/detect-structure` → analyse `anonymized.xlsx`
- `POST /api/projects/{id}/structure` → sauvegarde `structure.json`
- `POST /api/corpus/{id}/detect-structure` → analyse `anonymized.xlsx` (docx → `{"skipped": true}`)
- `POST /api/corpus/{id}/structure` → sauvegarde `structure.json`

### Frontend partagé
- `renderStructureForm(formId, data, onConfirm, onBack)` — formulaire éditable commun
- `runStructureDetection({detectUrl, saveUrl, …})` — orchestration appel API + affichage
- Badge confiance : `high` (vert) / `medium` (orange) / `low` (rouge)
- Si Claude indisponible : formulaire vide, remplissage manuel (non bloquant)
- Colonnes forcées en majuscules à la saisie

### Flux finaux
```
PAS    : Upload → Anonymisation (étape 1/3) → Structure Claude (2/3) → Résultat (3/3)
Corpus : Upload → Wizard (1/5) → Anonymisation (2/5) → Structure Claude (3/5) → Infos (4/5) → Save (5/5)
```
