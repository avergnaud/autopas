# Prompt 27

## Feature : Politiques sécurité — passage de multi-PDF à un seul fichier Markdown

### Besoin

- **Comportement actuel** : les Politiques sécurité sont plusieurs fichiers `.docx` (un par politique), chacun dans un répertoire UUID sous `data/policies/`, avec un workflow d'anonymisation.
- **Comportement attendu** : une seule fichier Markdown (`.md`) qui peut être uploadé (et ré-uploadé si les politiques changent).

### Changements à implémenter

1. **Backend** (`app/api/web.py`) : remplacer les endpoints multi-fichiers UUID par 3 endpoints simples :
   - `POST /api/policies` — upload/remplacement du fichier `.md` (chemin fixe `data/policies/politiques.md`)
   - `GET /api/policies` — info + contenu du fichier courant
   - `DELETE /api/policies` — suppression du fichier

2. **Frontend** (`web/private.html`) : remplacer le wizard upload+anonymisation par un simple bouton d'upload avec indicateur de statut (fichier présent / absent, date de mise à jour).

3. **Stockage** : `data/policies/politiques.md` (chemin fixe, pas de répertoires UUID).

4. **Génération** : non traité dans ce prompt (à voir plus tard pour l'injection dans le prompt Claude).
