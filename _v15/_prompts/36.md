# Prompt 36 — Upload et anonymisation du contrat

## Objectif

Permettre à l'utilisateur d'associer un contrat (fichier `.docx`) à un questionnaire, dans les deux wizards existants : **"Analyser un nouveau PAS"** (onglet PAS) et **"Base de connaissances"** (onglet Corpus).

Le contrat est optionnel. S'il est fourni :
- ses métadonnées PII sont fusionnées avec celles du questionnaire dans l'étape d'anonymisation, afin que l'utilisateur puisse définir les remplacements en une seule passe ;
- il est anonymisé avec le même mapping que le questionnaire ;
- dans le wizard PAS uniquement, le texte anonymisé est injecté comme contexte supplémentaire dans l'appel Claude lors de la génération ;
- dans le wizard Corpus, le fichier anonymisé est persisté dans le dossier de l'entrée corpus (aux côtés de `anonymized.xlsx`, `structure.json`, etc.).

## Architecture générale

L'upload du contrat se fait en **deux requêtes séquentielles** depuis le frontend :
1. Upload du questionnaire (endpoint existant, inchangé).
2. Upload du contrat `.docx` vers un **nouvel endpoint dédié** (si l'utilisateur a sélectionné un contrat).

L'anonymisation du contrat est déclenchée automatiquement par les endpoints `/anonymize` existants, qui vérifieront si `contract_original.docx` est présent dans le répertoire et, le cas échéant, l'anonymiseront avec le même `raw_mapping`.

## 2. Backend — `app/services/response_generator.py`

### 2.1 Nouvelle fonction `_read_contract_text`

Ajouter une fonction pour lire `contract_anonymized.docx` depuis le dossier projet :

```python
def _read_contract_text(project_dir: Path) -> str | None:
    """Read and return the anonymized contract text, or None if absent.

    Args:
        project_dir: Path to the project directory.

    Returns:
        Contract text (capped at 50 000 chars) or None.
    """
    contract_path = project_dir / "contract_anonymized.docx"
    if not contract_path.exists():
        return None

    from docx import Document as DocxDocument

    doc = DocxDocument(contract_path)
    parts: list[str] = []

    for para in doc.paragraphs:
        if para.text.strip():
            parts.append(para.text.strip())

    for table in doc.tables:
        for row in table.rows:
            cells = [c.text.strip() for c in row.cells if c.text.strip()]
            if cells:
                parts.append(" | ".join(cells))

    text = "\n".join(parts)
    if len(text) > 50_000:
        text = text[:50_000] + "\n[... tronqué]"

    logger.info("Contract text loaded: %d chars", len(text))
    return text if text.strip() else None
```

## 6. Contraintes à respecter

- **R-01** : ne jamais modifier `contract_original.docx`. L'anonymisation produit `contract_anonymized.docx` séparément.
- **KISS** : si `contract_original.docx` est absent, aucun chemin de code modifié ne lève d'erreur — tout se comporte comme avant.
- **Limite de taille** : `_read_contract_text` tronque à 50 000 caractères, comme `_read_corpus_entry_docx`.
- **`anyio`** : tous les appels bloquants (lecture docx, anonymisation) dans les endpoints `async` doivent être wrappés avec `await anyio.to_thread.run_sync(lambda: ...)`.
