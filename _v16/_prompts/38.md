# Prompt 38 — Contraintes métier déduites du cadrage

## Contexte

Comparaison entre un fichier rempli par un humain (`AVE.QUESTIONNAIRE FOURNISSEUR AESIO MUTUELLE 2026 (1).xlsx`) et le fichier généré par l'app (`PAS_rempli_6dd38b9b.xlsx`).

Cellule G40 (question A 3.2 — analyse de risque du SI délivrant le service) :
- **Humain** : répond N/A, car en AT avec poste CLIENT, le FOURNISSEUR n'a pas de SI propre.
- **App** : génère une réponse longue qui suppose un "dispositif à engagement" et invente une référence interne (`PLS_M07_11`).

## Problème

Le prompt envoyé à Claude ne lui donne pas les contraintes logiques déductibles du cadrage. Claude hallucine un contexte projet (dispositif à engagement, SI FOURNISSEUR, politique interne) incompatible avec le cadrage réel.

## Solution : bloc "Contraintes métier" généré dynamiquement

Construire un bloc de texte à partir des valeurs du cadrage, injecté dans le prompt système envoyé à Claude. Ce bloc explicite ce qui est **hors périmètre** selon le contexte, et indique à Claude de répondre N/A pour les questions concernées.

### Règles à implémenter

| Champ cadrage | Valeur | Contrainte à injecter |
|---|---|---|
| `type_prestation_base` | `Assistance Technique` | Pas de dispositif à engagement, pas de projet forfait, pas de livraison/recette. Les intervenants travaillent en régie sur le SI du CLIENT. |
| `hebergement_donnees` | `SI CLIENT` | Le FOURNISSEUR n'héberge aucune donnée client. Questions sur infra/hébergement/DR/PCA côté FOURNISSEUR → N/A. |
| `poste_travail` | `CLIENT` | Le FOURNISSEUR ne fournit pas de postes de travail. Questions MDM, AV, chiffrement disque, durcissement poste FOURNISSEUR → N/A. |
| `sous_traitance_rgpd` | `Non` | Aucun sous-traitant impliqué dans la prestation. Questions gestion sous-traitants RGPD → N/A. |

### Format du bloc injecté (exemple pour le cadrage AESIO)

```
## Contraintes liées au contexte de la prestation

- Type de prestation : Assistance Technique. Les intervenants travaillent en régie sur le SI du CLIENT. Il n'existe pas de dispositif à engagement, de projet forfait, de livraison ou de recette. Ne pas supposer l'existence d'un SI FOURNISSEUR déployé chez le client.
- Hébergement : les données résident sur le SI CLIENT. Le FOURNISSEUR n'héberge aucune donnée client. Les questions portant sur l'infrastructure, l'hébergement, le DR ou le PCA côté FOURNISSEUR sont sans objet.
- Poste de travail : fourni par le CLIENT. Le FOURNISSEUR ne fournit pas de matériel informatique. Les questions portant sur la sécurité des postes FOURNISSEUR (MDM, AV, chiffrement disque) sont sans objet.
- Sous-traitance RGPD : aucune sous-traitance impliquée. Ne pas supposer l'existence de sous-traitants.

Pour toute question portant sur un périmètre couvert par ces contraintes, répondre N/A avec une justification d'une phrase expliquant pourquoi la question est sans objet dans ce contexte.
```

## Implémentation

1. Créer une fonction `build_constraints_block(cadrage: dict) -> str` dans `response_generator.py` (ou un module dédié).
2. Injecter le bloc retourné dans le prompt système, après le contexte de cadrage existant.
3. La fonction construit le bloc conditionnellement selon les valeurs du cadrage — aucune règle en dur si le champ a une valeur différente.
