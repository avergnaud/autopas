# Prompt 08

Two successive corruption issues found during roundtrip testing.

## Issue 1 — External links

Excel repair dialog:
  "Enregistrements réparés: Référence de formule externe dans la partie
   /xl/externalLinks/externalLink1.xml"

The file contains external links (references to other Excel files).
openpyxl cannot round-trip them correctly → broken XML → repair dialog.

Fix: `load_workbook(working_path, keep_links=False)`
Discards external link definitions, keeps cached cell values.

## Issue 2 — Named ranges (all variants)

Excel repair dialog (second and third roundtrip tests):
  "Enregistrements supprimés: Plage nommée dans la partie /xl/workbook.xml"

openpyxl cannot reliably serialize all defined name variants (external references,
sheet-scoped names, complex formula syntax). Filtering only by '[' in the value
was not sufficient — local named ranges also triggered the error.

Attempts 1-2: `del wb.defined_names[name]` / `wb.defined_names._dict` — both wrong.
Inspecting the installed openpyxl source revealed:
- `wb.defined_names` is a `DefinedNameDict` (plain dict subclass) — no `_dict` attribute
- `_xlnm.Print_Area` / `_xlnm.Print_Titles` are stored as worksheet attributes and
  serialized into workbook.xml at save time, separately from `wb.defined_names`

Inspecting workbook/_writer.py `write_names()` revealed THREE sources of defined names:
  1. `wb.defined_names` — workbook-level named ranges
  2. `ws.defined_names` — **sheet-level** named ranges (missed in all previous attempts)
  3. `ws.print_titles` / `ws.print_area` — per-worksheet print settings

Fix: clear all three on every worksheet.

## What was changed

- `app/api/web.py` — `_openpyxl_roundtrip()`:
  1. `load_workbook(..., keep_links=False)` — strip external links
  2. `wb.defined_names.clear()` — workbook-level named ranges
  3. `ws.defined_names.clear()` + clear print settings on each worksheet
