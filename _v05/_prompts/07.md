# Prompt 07

Sub-step A before anonymization: test the openpyxl round-trip (open + save, no changes).

`shutil.copy2` (raw bytes) is proven safe. But the first time openpyxl opens and saves the
file is a separate risk: charts, pivot tables, named ranges, etc. can silently break.
We need to verify this before building any transformation on top of openpyxl.

Please implement sub-step A:
- openpyxl open working.xlsx + save unchanged to roundtrip.xlsx
- provide a download link for roundtrip.xlsx
- the user compares working.xlsx vs roundtrip.xlsx to check for corruption

Do NOT overwrite working.xlsx.

Also persist the corresponding prompt in _v05/_prompts/07.md.

## What was implemented

### Modified files
- `app/api/web.py`:
  - added `_openpyxl_roundtrip(working_path, roundtrip_path)` — synchronous, pure open/save identity via openpyxl
  - added `POST /api/projects/{project_id}/roundtrip` — runs the round-trip in a thread (anyio), saves to `roundtrip.xlsx`
  - added `GET /api/projects/{project_id}/roundtrip` — downloads `roundtrip.xlsx`
- `web/private.html`:
  - after upload: shows both a "Télécharger working.xlsx" link and a "Tester le round-trip openpyxl" button
  - after round-trip: shows a "Télécharger roundtrip.xlsx" link

### Files in project directory after this step
- `original.xlsx` — untouched original (raw bytes from upload)
- `working.xlsx` — raw copy via shutil.copy2 (also untouched)
- `roundtrip.xlsx` — result of openpyxl open+save (what to compare against)
