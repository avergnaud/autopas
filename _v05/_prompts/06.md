# Prompt 06

Current working implementation is in the _v05 folder.

For this baby step, I want to work on one specific feature: xlsx processing.
Indeed, the xlsx files will be provided by our clients.
For each file,
- the user will upload the file
- the wizard will analyze the file and check the file's structure with the user
- during the wizard, the user may choose to remove some tabs from the file
- server side (ubuntu), the file will go through anonymization (both for metadatas and content).
- server side, some tabs may be removed
- server side, the Claude API may answer the various questions (meaning writing in the file...)
- then the modified file will be made available for the user to download...

During all this processing, the xlsx may become corrupted.

So I want to be extra careful and implement one very small step at a time.
And for each step, check the xlsx file does not get corrupted.

We are going to do it in paranoid mode.

So please:
- just implement the following: upload, copy file (with new name), provide link to download the copied file
- write the corresponding prompt into _v05/_prompts/06.md

## What was implemented

### New files
- `app/api/__init__.py` — package marker
- `app/api/web.py` — two endpoints:
  - `POST /api/upload` — accepts an `.xlsx` file, creates a UUID project directory under `data/projects/{project_id}/`, saves `original.xlsx`, copies it to `working.xlsx`, returns `project_id`, `filename`, `download_url`
  - `GET /api/projects/{project_id}/working` — serves `working.xlsx` as a file download

### Modified files
- `app/main.py` — includes `web_router` from `app.api.web`
- `web/private.html` — replaced placeholder with an upload form; after upload, shows project ID and a download link for the working copy

### Design decisions
- Original file is **never modified** — all subsequent steps will work on `working.xlsx`
- No xlsx parsing at this stage — the file is copied as raw bytes (`shutil.copy2`), guaranteeing zero corruption risk
- Auth required on both endpoints (`get_current_user`)
- Only `.xlsx` accepted (extension check on upload)
