# Prompt 09

Roundtrip bug: data-validation dropdowns in column F broken after openpyxl round-trip.

## Root cause

In the raw workbook.xml the file contained **duplicate definedName entries** for the same
name (e.g. `Cotation`):

```xml
<!-- LOCAL — sheet-scoped, points to cells inside this workbook ✓ -->
<definedName name="Cotation" localSheetId="0">'PESA SSI Presta FR'!$A$7:$A$10</definedName>

<!-- WORKBOOK-LEVEL — broken external ref (keep_links=False turns [n] into #REF!) ✗ -->
<definedName name="Cotation">#REF!</definedName>
```

`openpyxl`'s `DefinedNameDict` is keyed by name only. When reading, the second entry
(`#REF!`, workbook-level) overwrites the first (valid local one). Our `wb.defined_names.clear()`
then removed everything, leaving the `Cotation` named range entirely absent from the output
file. Data-validation cells referencing `Cotation` kept their existing string values but lost
the dropdown picker.

## Fix

Two-step approach in `_openpyxl_roundtrip()`:

1. **Before** loading with openpyxl, parse the raw XML directly (`zipfile` + `xml.etree`)
   and extract every `definedName` that is safe to keep:
   - no `[n]` external-workbook reference
   - no `#REF!`
   - not a `_xlnm.*` built-in
   - not hidden

2. After the usual clear-all, **re-inject** those safe names as proper `DefinedName` objects
   (with `localSheetId` preserved) via `wb.defined_names.add()`.

New helper: `_safe_local_defined_names(xlsx_path)` in `app/api/web.py`.

## Files changed

- `app/api/web.py`:
  - added imports: `re`, `xml.etree.ElementTree`, `zipfile`, `DefinedName`
  - added `_XLSX_NS` constant
  - added `_safe_local_defined_names()` helper
  - modified `_openpyxl_roundtrip()`: call helper before load, re-inject after clear

## Result (verified on QUESTIONNAIRE FOURNISSEUR AESIO MUTUELLE 2026.xlsx)

Original: 32 named ranges (30 external/broken, 3 safe local: Cotation, Avis, Quoting)
Roundtrip output: 3 named ranges, all local, correct XML with `localSheetId="0"`.
Data-validation dropdowns on F32:F122 and F137/F142 functional again.
