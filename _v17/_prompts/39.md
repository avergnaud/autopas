# Prompt 39 — Sauvegarde et téléchargement des prompts Claude

## Demande

Les logs serveur affichent : `"Calling Claude for responses — prompt length: ~96345 chars"`

Nouvelle fonctionnalité : le prompt envoyé à l'API Claude doit être sauvegardé côté serveur, et le dernier écran "Génération terminée avec succès !" doit afficher un lien de téléchargement de ce prompt.

## Implémentation

### `app/services/response_generator.py`
- Après la construction de `user_prompt_resp`, écriture de `{project_dir}/prompt_debug.txt` avec les sections :
  - `=== SYSTEM (responses) ===`
  - `=== USER (responses) ===`
- Après la construction de `user_prompt_attn`, ajout en append dans le même fichier :
  - `=== SYSTEM (attention) ===`
  - `=== USER (attention) ===`

### `app/api/web.py`
- Nouvel endpoint `GET /api/projects/{project_id}/prompt`
- Sert `prompt_debug.txt` en `text/plain; charset=utf-8`
- Même pattern d'auth que `/output` et `/attention`
- Filename de téléchargement : `prompt_debug_{project_id[:8]}.txt`

### `web/private.html`
- Nouvelle download card "Prompts Claude (debug)" avec `id="pas-prompt-link"`
- Description : "Texte brut des prompts envoyés à l'API Claude lors de la génération."
- Le JS assigne `href = /api/projects/${_pasProjectId}/prompt` lors du passage en statut `completed`
