# Prompt 19 — Wizard de cadrage (onglet PAS)

## Objectif

Ajouter le wizard de questions de cadrage dans le workflow de l'onglet **Analyser un nouveau PAS**, après la validation de la structure.

Le wizard est déjà implémenté (classe `QuestionWizard`) et fonctionne dans l'onglet Corpus. Il faut le brancher dans le workflow PAS.

## Ce qu'il faut faire

### Backend — `app/api/web.py`

Deux nouveaux endpoints :

**1. `GET /api/projects/{project_id}/cadrage`**
- Charge `project.json` et retourne `{"answers": data["cadrage"]}` (null si pas encore rempli)
- Protégé par `get_current_user`

**2. `POST /api/projects/{project_id}/cadrage`**
- Body Pydantic : `{"answers": dict}`
- Sauvegarde `answers` dans `project.json["cadrage"]`
- Passe le status à `"cadrage_done"` via `update_project`
- Retourne `{"status": "cadrage_done"}`
- Protégé par `get_current_user`

### Frontend — `web/private.html`

**Nouvelle section HTML dans le tab PAS** (entre structure et done) :

```html
<!-- 4. Cadrage -->
<div id="pas-cadrage-section" style="display:none;">
  <p class="step-hint">Etape 3 / 3 — Questions de cadrage</p>
  <div id="pas-cadrage-container"></div>
  <p class="error-msg" id="pas-cadrage-error"></p>
</div>
```

**Modifier `pasShowSection`** pour inclure `pas-cadrage-section` dans la liste des sections.

**Modifier `pasDetectStructure` → `onConfirm`** :
Au lieu d'aller à `pas-done-section`, appeler `pasStartCadrage()`.

**Nouvelle fonction `pasStartCadrage()`** :
1. Fetch `GET /api/questions` pour obtenir la liste des questions
2. Instancier `QuestionWizard` dans `pas-cadrage-container`
3. `onComplete(answers)` → POST `/api/projects/{_pasProjectId}/cadrage` avec `{answers}`
4. Si succès → `pasShowSection('pas-done-section')`

**Modifier `pas-done-section`** :
- Changer le message de succès : "Cadrage enregistré."
- Ajouter une note : "La génération des réponses sera disponible prochainement."
- Garder le download anonymized.xlsx et le tableau de mapping (utiles comme référence).

**Mettre à jour les step hints** dans les sections existantes :
- Anonymisation : "Etape 1 / 3"
- Structure : "Etape 2 / 3"
- Cadrage : "Etape 3 / 3" ← nouvelle

## Ce qui ne change pas

- La classe `QuestionWizard` : réutilisée telle quelle (pas de modification)
- L'endpoint `GET /api/questions` : existe déjà, on le réutilise
- Le workflow Corpus : inchangé
- `project_manager.update_project` : utilisé tel quel

## États projet après cette itération

```
created → (anonymize) → (structure) → cadrage_done
```
