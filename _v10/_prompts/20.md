# Prompt 20 — Sélection manuelle du corpus (onglet PAS)

## Objectif

Après le cadrage, afficher à l'utilisateur la liste des entrées corpus classées par score de pertinence, avec des cases à cocher. L'utilisateur valide les fichiers à utiliser comme contexte pour Claude. La sélection est sauvegardée dans `project.json`.

## Nouveau service — `app/services/reference_selector.py`

Fonction principale :

```python
def score_corpus_entries(cadrage: dict, entries: list[dict]) -> list[dict]:
```

- Pour chaque entrée corpus, calcule un score de correspondance avec les réponses de cadrage du projet.
- Retourne la liste triée par score décroissant, avec le champ `score` ajouté à chaque entrée.

**Règles de scoring** (valeurs cumulatives, pas de maximum imposé) :

| Condition | Points |
|-----------|--------|
| `type_prestation` identique | +3 |
| `secteur_client` identique | +2 |
| `hebergement_donnees` identique | +1 |
| `expertise_atlassian` identique | +1 |
| `sous_traitance_rgpd` identique | +1 |
| `lieu_travail` — au moins un lieu en commun | +1 |
| `poste_travail` identique | +1 |

Les champs manquants (null ou absent) ne pénalisent pas, ils sont simplement ignorés.

Pour les champs `type_prestation`, les métadonnées corpus stockent `type_prestation` (calculé dans `_resolve_type_prestation` en `web.py`). Les réponses cadrage contiennent `type_prestation_base` et `type_prestation_detail`. Considérer qu'il y a correspondance si `cadrage["type_prestation_base"]` est inclus dans `entry["type_prestation"]` (comparaison insensible à la casse).

## Backend — `app/api/web.py`

Deux nouveaux endpoints :

**1. `GET /api/projects/{project_id}/corpus-selection`**
- Charge `project.json` → récupère `cadrage`
- Si `cadrage` est null → 400 "Cadrage non effectué."
- Charge toutes les entrées corpus (réutiliser la logique de `list_corpus`)
- Appelle `score_corpus_entries(cadrage, entries)`
- Retourne `{"entries": [...]}` (liste triée par score décroissant, chaque entrée a `corpus_id`, `filename`, `type_prestation`, `secteur_client`, `nb_etp`, `date_remplissage`, `score`)

**2. `POST /api/projects/{project_id}/corpus-selection`**
- Body Pydantic : `{"selected_corpus_ids": list[str]}`
- Valide que la liste est non vide → sinon 400 "Sélectionnez au moins une entrée."
- Valide que chaque `corpus_id` existe dans `CORPUS_DIR` → sinon 400
- Sauvegarde dans `project.json["selected_corpus"]` la liste des ids
- Passe le status à `"corpus_selected"` via `update_project`
- Retourne `{"status": "corpus_selected", "count": N}`

### Modifications JS

**`pasShowSection`** : ajouter `'pas-corpus-section'` dans la liste.

**`pasStartCadrage` → `onComplete`** : au lieu d'aller à `pas-done-section`, appeler `pasLoadCorpusSelection()`.

**Nouvelle fonction `pasLoadCorpusSelection()`** :
1. `pasShowSection('pas-corpus-section')`
2. Afficher le spinner, masquer `pas-corpus-ready`
3. Fetch `GET /api/projects/{_pasProjectId}/corpus-selection`
4. En cas d'erreur → afficher dans `pas-corpus-error`
5. Rendre la liste dans `pas-corpus-entries` : pour chaque entrée, une `.corpus-select-row` avec checkbox (pré-cochée si score > 0, décochée si score === 0), titre = filename, meta = type_prestation · secteur_client · nb_etp ETP · date, badge score
6. Si corpus vide → message "Aucun fichier dans la base de connaissances. Ajoutez des fichiers dans l'onglet Corpus."
7. Masquer spinner, afficher `pas-corpus-ready`

**Nouvelle fonction `pasCorpusToggleAll(checked)`** : coche/décoche toutes les checkboxes dans `pas-corpus-entries`.

**Nouvelle fonction `pasConfirmCorpus()`** :
1. Récupère les `corpus_id` cochés (attribut `data-corpus-id` sur chaque checkbox)
2. Si aucun coché → `pas-corpus-error` = "Sélectionnez au moins une entrée."
3. Fetch `POST /api/projects/{_pasProjectId}/corpus-selection` avec `{selected_corpus_ids: [...]}`
4. Si erreur → afficher dans `pas-corpus-error`
5. Si succès → `pasShowSection('pas-done-section')` + remplir tableau mapping + lien download (logique déjà dans `pasStartCadrage.onComplete` → déplacer ici)
