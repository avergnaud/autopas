# Prompt 22 — Correction articles FOURNISSEUR/CLIENT (traitements A + B)

## Contexte

Après la dé-anonymisation, certaines réponses générées par Claude contiennent des
formulations grammaticalement incorrectes :

- `"le Sun Microsystems"` (article défini inutile devant un nom propre)
- `"au Enron"` → devrait être `"à Enron"` (élision devant voyelle)
- `"du Enron"` → devrait être `"de Enron"`

Claude générait correctement `"le FOURNISSEUR"` / `"du CLIENT"` / `"au CLIENT"`,
mais la dé-anonymisation remplace mécaniquement sans corriger le contexte grammatical.

**Règles définies (français) :**
- FOURNISSEUR et CLIENT sont toujours masculins.
- Ne jamais précéder FOURNISSEUR/CLIENT d'un article défini : pas de `"le"`, pas de `"la"`.
- Ne jamais écrire `"du FOURNISSEUR"` ni `"du CLIENT"` : utiliser `"de FOURNISSEUR"` / `"de CLIENT"`.
- Ne jamais écrire `"au FOURNISSEUR"` ni `"au CLIENT"` : utiliser `"à FOURNISSEUR"` / `"à CLIENT"`.

---

## Traitement "français A" — Prompt engineering (system_response.txt et system_attention.txt)

Ces règles s'appliquent pour les documents en **français**. Un équivalent sera prévu pour l'anglais.

Ajouter dans `data/config/prompts/system_response.txt`, dans la section `CONSIGNES` :

```
- FOURNISSEUR et CLIENT sont des termes de substitution (placeholders), qui seront remplacés plus tard par de vrais noms d'entreprises.
- FOURNISSEUR et CLIENT sont des noms propres masculins (documents en français).
  Ne jamais les faire précéder d'un article défini ("le", "la").
  Écrire directement "FOURNISSEUR" et "CLIENT" sans article.
- Ne jamais écrire "du FOURNISSEUR" ni "du CLIENT" : écrire "de FOURNISSEUR" et
  "de CLIENT".
- Ne jamais écrire "au FOURNISSEUR" ni "au CLIENT" : écrire "à FOURNISSEUR" et
  "à CLIENT".
```

Même ajout dans `data/config/prompts/system_attention.txt`.

## Traitement "français B" — Post-traitement sur les tokens (avant dé-anonymisation)

Le traitement a lieu **avant** la dé-anonymisation, directement sur les chaînes contenant
les tokens `FOURNISSEUR` et `CLIENT`. Cela permet un remplacement exact sur des chaînes
connues, sans détection de voyelle.

Ces règles s'appliquent pour les documents en **français**. Un équivalent sera prévu pour l'anglais.

**Règles de remplacement (regex, dans cet ordre) :**

| Avant (token présent) | Après |
|---|---|
| `au FOURNISSEUR` | `à FOURNISSEUR` |
| `au CLIENT` | `à CLIENT` |
| `du FOURNISSEUR` | `de FOURNISSEUR` |
| `du CLIENT` | `de CLIENT` |
| `le FOURNISSEUR` | `FOURNISSEUR` |
| `le CLIENT` | `CLIENT` |
| `la FOURNISSEUR` | `FOURNISSEUR` |
| `la CLIENT` | `CLIENT` |

Pattern regex généralisé (insensible à la casse pour l'article) :

```python
import re

_TOKENS = r'(?:FOURNISSEUR|CLIENT)'

_FRENCH_ARTICLE_FIXES = [
    (re.compile(r'\bau\s+(' + _TOKENS + r')\b'),   r'à \1'),
    (re.compile(r'\bdu\s+(' + _TOKENS + r')\b'),   r'de \1'),
    (re.compile(r'\b[Ll]e\s+(' + _TOKENS + r')\b'), r'\1'),
    (re.compile(r'\b[Ll]a\s+(' + _TOKENS + r')\b'), r'\1'),
]

def fix_french_token_articles(text: str) -> str:
    """Corrige les articles contractés/définis devant FOURNISSEUR/CLIENT (avant dé-anonymisation)."""
    for pattern, repl in _FRENCH_ARTICLE_FIXES:
        text = pattern.sub(repl, text)
    return text
```

### Point d'intégration dans response_generator.py

Appliquer `fix_french_token_articles` :

1. Sur chaque `response` du JSON retourné par Claude, **avant** `write_responses` et **avant**
   `deanonymize_xlsx`.
2. Sur le texte des points d'attention, **avant** `deanonymize_text`.

Ainsi la dé-anonymisation reçoit un texte déjà corrigé (`"à CLIENT"`, `"de FOURNISSEUR"`, etc.)
et produit un résultat grammaticalement correct (`"à Enron"`, `"de Sun Microsystems"`, etc.).
