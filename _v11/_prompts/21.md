# Prompt 21 — Génération des réponses (cœur métier)

## Objectif

Implémenter la **génération complète** : à partir d'un projet au status `corpus_selected`,
appeler l'API Claude pour pré-remplir le questionnaire xlsx, générer les points d'attention,
dé-anonymiser le résultat, et proposer le téléchargement.

État final du workflow :
```
created → anonymized → structure_confirmed → cadrage_done → corpus_selected → generating → completed
                                                                                          └→ error
```

## 2. Nouveau service — `app/services/parser_xlsx.py`

Deux fonctions publiques :

### `read_questions(xlsx_path, structure) -> list[dict]`

Lit le xlsx avec openpyxl (`read_only=True, data_only=True`).
Utilise les champs `selected_sheet`, `first_data_row`, `col_id`, `col_question` de `structure`.
Retourne `[{"row": int, "question_id": str, "question_text": str}]` — une entrée par ligne
non vide dans la colonne question, à partir de `first_data_row`.
Si `col_id` est absent : `question_id = str(row_number)`.
Ignorer les lignes où `question_text` est vide.

Conversion col lettre → index openpyxl 1-based : utiliser `openpyxl.utils.column_index_from_string`.

### `write_responses(source_path, dest_path, structure, responses) -> None`

`responses = [{"question_id": str, "response": str}]`

1. `shutil.copy2(source_path, dest_path)` — copie source sans le modifier (R-01)
2. Extraire les defined names sûrs avec `safe_local_defined_names(dest_path)` (import depuis anonymizer)
3. Ouvrir avec `load_workbook(dest_path, keep_links=False, rich_text=False)`
4. Re-injecter les defined names (pattern identique à `_openpyxl_roundtrip` dans web.py)
5. Construire un dict `{question_id: response}` depuis `responses`
6. Itérer les lignes depuis `first_data_row` :
   - Récupérer `question_id` depuis `col_id` (ou `str(row_number)` si absent)
   - Si `question_id` dans le dict → écrire la réponse dans `col_response` via `ws.cell(row, col).value`
7. `wb.save(dest_path)` ; `wb.close()`

## 3. Dé-anonymisation dans `app/services/anonymizer.py`

Ajouter la fonction :

### `deanonymize_xlsx(source_path, dest_path, mapping) -> None`

`mapping` est `{original: token}` (tel que stocké dans `anonymized_map.json`).
L'inverse est `{token: original}`.

Appelle simplement `anonymize_xlsx(source_path, dest_path, {v: k for k, v in mapping.items()})`.
(La fonction existante gère déjà le tri par longueur décroissante — indispensable pour éviter
les remplacements partiels dans le sens inverse aussi.)

Ajouter aussi :

### `deanonymize_text(text, mapping) -> str`

Remplace les tokens par les originaux dans une chaîne de texte (pour les points d'attention).

```python
def deanonymize_text(text: str, mapping: dict[str, str]) -> str:
    for token, original in sorted(mapping.items(), key=lambda x: len(x[1]), reverse=True):
        text = text.replace(token, original)
    return text
```

Note : ici on trie par longueur de la **valeur** (original) décroissante pour éviter les
remplacements partiels dans la direction inverse.

## 4. Nouveau service — `app/services/response_generator.py`

Fonctions internes et une fonction publique `run_generation(project_id: str) -> None`.

### Helpers internes

**`_resolve_verbosity(cadrage, config) -> dict`**
Retourne `{"level": int, "label": str, "max_words": int}` à partir de
`cadrage.get("verbosity", "Standard")` et de la config.

Mapping label → level : `{"Concis": 1, "Standard": 2, "Détaillé": 3}`.
Fallback sur level 2 si inconnu.

**`_format_cadrage(cadrage) -> str`**
Formate les réponses de cadrage en texte lisible pour Claude.
Inclure tous les champs non vides. Exclure `verbosity`.
Format : `"- Clé : valeur\n..."`.

**`_read_corpus_entry(corpus_dir) -> str`**
Lit le contenu d'un fichier corpus pour l'inclure comme exemple.

- Charger `metadata.json` → récupérer `filename`, `format`
- Si format `xlsx` et `structure.json` existe :
  - `load_workbook(anonymized.xlsx, read_only=True, data_only=True)`
  - Utiliser la structure pour extraire (question_id, question_text, response_text)
  - Format par ligne : `"ID: {id} | Question: {q} | Réponse: {r}"`
  - Ignorer les lignes sans réponse
  - Max 150 lignes
- Si format `docx` :
  - Lire avec `python-docx`, extraire texte des paragraphes + cellules de tableaux
  - Max 50 000 caractères
- Si `anonymized.{fmt}` absent : retourner `""`
- Préfixer par `f"=== {filename} ===\n"`

**`_build_user_prompt(cadrage, corpus_contents, questions, verbosity) -> str`**

```
=== CONTEXTE DE LA PRESTATION ===
{_format_cadrage(cadrage)}

=== EXEMPLES DE QUESTIONNAIRES DÉJÀ REMPLIS ===
{chaque corpus_content séparé par "\n\n"}

=== QUESTIONNAIRE À REMPLIR ===
Niveau de verbosité : {label} — maximum {max_words} mots par réponse.

{pour chaque question : "ID: {question_id}\nQuestion: {question_text}\n---"}
```

**`_call_claude_json(system_prompt, user_prompt, model, max_tokens) -> dict`**
Appelle `anthropic.Anthropic().messages.create(...)` et parse le JSON retourné.
Strip éventuel bloc markdown ` ```json ... ``` `.

### Fonction principale `run_generation(project_id: str) -> None`

Appelée en BackgroundTask (s'exécute dans un thread pool — appels bloquants OK).

Wrapper :
```python
def run_generation(project_id: str) -> None:
    try:
        _do_generation(project_id)
    except Exception as exc:
        logger.exception("Generation failed for %s", project_id)
        try:
            project_manager.update_project(
                project_id, status="error", error_message=str(exc)
            )
        except Exception:
            pass
```

**`_do_generation(project_id: str) -> None`** :

```
project_dir = PROJECTS_DIR / project_id

1. Charger project.json → cadrage, selected_corpus, structure
   Vérifier que ces 3 champs sont présents.

2. update_project(status="generating", progress_step="Chargement des références...")

3. Charger config (api_key, model, max_tokens, verbosity config, reference.max_files)
   Vérifier ANTHROPIC_API_KEY.

4. Résoudre la verbosité → verbosity dict

5. Lire le contenu de chaque corpus sélectionné (selected_corpus ids)
   → liste corpus_contents (str)
   Limiter à reference.max_files entrées.

6. update_project(progress_step="Lecture du questionnaire...")

7. Charger structure.json depuis project_dir
   Appeler parser_xlsx.read_questions(project_dir / "anonymized.xlsx", structure)
   → questions list

8. update_project(progress_step="Génération des réponses (appel Claude)...")

9. Charger system_response.txt depuis prompts dir
   Construire user_prompt via _build_user_prompt(cadrage, corpus_contents, questions, verbosity)
   Appeler _call_claude_json(system_prompt, user_prompt, model, max_tokens)
   Extraire responses = result["responses"]  (list[dict])

10. update_project(progress_step="Écriture des réponses...")

11. parser_xlsx.write_responses(
        source_path = project_dir / "anonymized.xlsx",
        dest_path   = project_dir / "output_anon.xlsx",
        structure   = structure,
        responses   = responses,
    )

12. update_project(progress_step="Génération des points d'attention...")

13. Charger system_attention.txt
    Construire user_attention_prompt :
    "=== CONTEXTE ===\n{cadrage formaté}\n\n=== QUESTIONNAIRE REMPLI ===\n{questions + réponses}"
    Appeler _call_claude_json(system_attention, user_attention_prompt, model, max_tokens)
    Extraire attention_points = result["attention_points"]  (list[dict])

14. update_project(progress_step="Dé-anonymisation...")

15. Charger anonymized_map.json
    anonymizer.deanonymize_xlsx(
        project_dir / "output_anon.xlsx",
        project_dir / "output.xlsx",
        mapping
    )
    Supprimer output_anon.xlsx

16. Formater les points d'attention en Markdown + dé-anonymiser le texte :
    Pour chaque point d'attention, appliquer deanonymize_text sur description et recommendation.
    Sauvegarder project_dir / "attention.md"

17. update_project(
        status="completed",
        progress_step="Terminé.",
        error_message=None,
    )

18. logger.info("Generation completed for project %s", project_id)
```

**Format du Markdown `attention.md` :**
```markdown
# Points d'attention

## 1. [CATEGORIE] ID_QUESTION

Description du point d'attention.

**Recommandation :** Texte de la recommandation.

---
```

## 5. Backend — `app/api/web.py`

Ajouter 4 endpoints et les imports nécessaires.

**Import à ajouter :**
```python
from fastapi import BackgroundTasks
from app.services.response_generator import run_generation
```

### `POST /api/projects/{project_id}/generate`

- Charger project.json ; vérifier `status in {"corpus_selected", "completed", "error"}`
  (permettre la re-génération depuis `completed` ou `error`)
  Sinon 400 "Le projet n'est pas dans un état permettant la génération."
- `update_project(status="generating", progress_step="Démarrage...", error_message=None)`
- `background_tasks.add_task(run_generation, project_id)`
- Return 202 : `{"status": "generating"}`

### `GET /api/projects/{project_id}/status`

- Charger project.json
- Return `{"status": proj["status"], "progress_step": proj.get("progress_step"), "error_message": proj.get("error_message")}`

### `GET /api/projects/{project_id}/output`

- Vérifier existence de `output.xlsx` dans project_dir
- Return `FileResponse` (media_type xlsx, filename = `f"PAS_rempli_{project_id[:8]}.xlsx"`)

### `GET /api/projects/{project_id}/attention`

- Vérifier existence de `attention.md` dans project_dir
- Return `FileResponse` (media_type `text/markdown`, filename = `"points_attention.md"`)

#### JS — nouvelles fonctions

```js
let _pasGenPoller = null;

async function pasStartGeneration() {
  document.getElementById('pas-gen-init-error').textContent = '';
  const btn = document.getElementById('pas-gen-start-btn');
  btn.disabled = true;
  try {
    const res = await fetch(`/api/projects/${_pasProjectId}/generate`, { method: 'POST' });
    if (!res.ok) {
      const err = await res.json();
      document.getElementById('pas-gen-init-error').textContent = err.detail || 'Erreur lors du démarrage.';
      btn.disabled = false;
      return;
    }
  } catch {
    document.getElementById('pas-gen-init-error').textContent = 'Erreur réseau.';
    btn.disabled = false;
    return;
  }
  document.getElementById('pas-gen-initial').style.display = 'none';
  document.getElementById('pas-gen-running').style.display = '';
  document.getElementById('pas-gen-step').textContent = 'Démarrage...';
  _pasGenPoller = setInterval(pasPollStatus, 3000);
}

async function pasPollStatus() {
  try {
    const res = await fetch(`/api/projects/${_pasProjectId}/status`);
    if (!res.ok) return;
    const data = await res.json();
    if (data.progress_step) document.getElementById('pas-gen-step').textContent = data.progress_step;

    if (data.status === 'completed') {
      clearInterval(_pasGenPoller); _pasGenPoller = null;
      document.getElementById('pas-gen-running').style.display = 'none';
      document.getElementById('pas-gen-completed').style.display = '';
      document.getElementById('pas-output-link').href = `/api/projects/${_pasProjectId}/output`;
      document.getElementById('pas-attention-link').href = `/api/projects/${_pasProjectId}/attention`;
    } else if (data.status === 'error') {
      clearInterval(_pasGenPoller); _pasGenPoller = null;
      document.getElementById('pas-gen-running').style.display = 'none';
      document.getElementById('pas-gen-error').style.display = '';
      document.getElementById('pas-gen-error-msg').textContent =
        data.error_message || 'Une erreur est survenue pendant la génération.';
    }
  } catch { /* ignore transient errors */ }
}

function pasRetryGeneration() {
  document.getElementById('pas-gen-error').style.display = 'none';
  document.getElementById('pas-gen-initial').style.display = '';
  document.getElementById('pas-gen-start-btn').disabled = false;
}

function pasRegenerateConfirm() {
  document.getElementById('pas-gen-completed').style.display = 'none';
  document.getElementById('pas-gen-initial').style.display = '';
  document.getElementById('pas-gen-start-btn').disabled = false;
}
```

## 7. Filesystem — structure finale d'un projet généré

```
data/projects/{uuid}/
  original.xlsx         ← jamais modifié
  working.xlsx          ← copie initiale (identique à original)
  anonymized.xlsx       ← version anonymisée (base de la génération)
  anonymized_map.json   ← table de correspondance {original: token}
  structure.json        ← structure confirmée
  project.json          ← état et métadonnées
  output_anon.xlsx      ← intermédiaire (supprimé après dé-anonymisation)
  output.xlsx           ← résultat final (réponses pré-remplies, dé-anonymisé)
  attention.md          ← points d'attention (Markdown, dé-anonymisé)
```
