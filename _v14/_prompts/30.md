# Prompt 30

## Contexte

Dans l'étape d'anonymisation (onglet PAS et onglet Corpus), il existe deux catégories de lignes :
1. **Lignes fixes** (non supprimables, readonly) : ajoutées par `addFixedKeywords()` — ex. Catamania → FOURNISSEUR.
2. **Lignes normales** (supprimables, éditables) : ajoutées via les suggestions ou le bouton "+ Ajouter une ligne".

## Changement demandé

### Nouvelle ligne pré-remplie « → CLIENT »

Ajouter automatiquement une ligne normale (supprimable, avec bouton poubelle) pré-remplie ainsi :
- **Mot-clé original** : chaîne vide `""` (champ vide, à remplir par l'utilisateur)
- **Remplacement** : `CLIENT`

Cette ligne doit apparaître juste après les lignes fixes, avant les suggestions extraites du fichier.
Elle est une ligne **normale** (pas `fixed`) : l'utilisateur peut la supprimer s'il le souhaite.

### Désactivation du bouton « Anonymiser → »

Le bouton "Anonymiser →" (id `pas-anon-btn` côté PAS, id `corpus-anon-btn` côté Corpus) doit être **désactivé** (`disabled`) tant qu'au moins une ligne possède un **champ "Mot-clé original" vide** (input `.kw-original` avec `value.trim() === ""`).

Dès que tous les champs "Mot-clé original" sont remplis (non vides), le bouton redevient actif.

Cette validation doit se faire **en temps réel** via un écouteur d'événement `input` sur le conteneur des lignes (`pas-kw-rows` / `corpus-anon-rows`), qui vérifie à chaque frappe si un champ original est vide.

### Ce qui NE change PAS

- Les lignes readonly (fixes) n'ont pas de champ `.kw-original` vide (elles ont toujours une valeur pré-remplie) → elles ne bloquent jamais le bouton.
- La validation existante dans `pasDoAnonymize()` / `corpusDoAnonymize()` (vérification que `keywords.length > 0`) reste inchangée.
- Le comportement du bouton "+ Ajouter une ligne" reste identique : il ajoute une ligne vide normale.

## Résumé des modifications à effectuer

### `web/private.html`

**Fonction `pasLoadAnonSuggestions()`** :
1. Après `addFixedKeywords('pas-kw-rows')`, ajouter une ligne normale avec `addKwRow('pas-kw-rows', '', 'CLIENT')`.
2. Brancher un écouteur `input` sur le conteneur `#pas-kw-rows` qui appelle une fonction `updateAnonBtn('pas-kw-rows', 'pas-anon-btn')` à chaque modification.
3. Appeler `updateAnonBtn('pas-kw-rows', 'pas-anon-btn')` une première fois après le rendu pour initialiser l'état du bouton (désactivé au départ car le champ original est vide).

**Fonction `corpusLoadAnonSuggestions()`** :
1. Même logique : après `addFixedKeywords('corpus-anon-rows')`, ajouter `addKwRow('corpus-anon-rows', '', 'CLIENT')`.
2. Brancher un écouteur `input` sur `#corpus-anon-rows` appelant `updateAnonBtn('corpus-anon-rows', 'corpus-anon-btn')`.
3. Appeler `updateAnonBtn('corpus-anon-rows', 'corpus-anon-btn')` une première fois pour initialiser.

**Nouvelle fonction partagée `updateAnonBtn(rowsContainerId, btnId)`** :
```javascript
function updateAnonBtn(rowsContainerId, btnId) {
  const hasEmpty = [...document.querySelectorAll(`#${rowsContainerId} .kw-original`)]
    .some(inp => !inp.readOnly && inp.value.trim() === '');
  document.getElementById(btnId).disabled = hasEmpty;
}
```
Note : on ignore les inputs `readonly` (lignes fixes) car ils ont toujours une valeur.

**Cas particulier — "+ Ajouter une ligne"** :
Quand l'utilisateur clique sur "+ Ajouter une ligne", une ligne vide est ajoutée → le bouton doit immédiatement se désactiver. Puisque l'écouteur `input` est sur le conteneur (event delegation), il ne se déclenche pas à l'ajout de la ligne. Il faut donc appeler `updateAnonBtn(...)` directement dans les `onclick` des boutons "+ Ajouter une ligne" :
- `onclick="addKwRow('pas-kw-rows','',''); updateAnonBtn('pas-kw-rows','pas-anon-btn')"`
- `onclick="addKwRow('corpus-anon-rows','',''); updateAnonBtn('corpus-anon-rows','corpus-anon-btn')"`

De même, quand une ligne est supprimée via le bouton poubelle, `updateAnonBtn` doit être rappelé. Modifier `addKwRow` pour que le bouton de suppression appelle `updateAnonBtn` après suppression :
```javascript
`<button class="btn-icon" title="Supprimer" onclick="this.closest('.kw-row').remove(); updateAnonBtn('${containerId}', ...)">&#x1F5D1;</button>`
```
Problème : `addKwRow` ne connaît pas l'id du bouton Anonymiser. Solution : passer le `btnId` en paramètre de `addKwRow`, ou utiliser une convention (le bouton est toujours dans le même formulaire parent). Solution plus simple : dans le `onclick` du bouton poubelle, après suppression, chercher dynamiquement le bouton Anonymiser dans le même formulaire parent. Ou encore plus simple : utiliser un `MutationObserver` sur le conteneur, ou appeler `updateAnonBtn` depuis un écouteur `click` sur le conteneur (en plus de `input`).

**Solution retenue pour la suppression** : ajouter un écouteur `click` sur le conteneur pour détecter les clics sur `.btn-icon` (bouton poubelle). Après suppression (via `setTimeout(0)` pour laisser le DOM se mettre à jour), appeler `updateAnonBtn`. Cela évite de passer des paramètres complexes dans `addKwRow`.

Exemple d'implémentation dans `pasLoadAnonSuggestions()` :
```javascript
const rowsEl = document.getElementById('pas-kw-rows');
rowsEl.addEventListener('input', () => updateAnonBtn('pas-kw-rows', 'pas-anon-btn'));
rowsEl.addEventListener('click', e => {
  if (e.target.closest('.btn-icon')) setTimeout(() => updateAnonBtn('pas-kw-rows', 'pas-anon-btn'), 0);
});
updateAnonBtn('pas-kw-rows', 'pas-anon-btn');
```
(Idem pour `corpus-kw-rows` / `corpus-anon-btn`.)
